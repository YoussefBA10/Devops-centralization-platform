# Phase 5: GitOps Directory Structure & Configuration

**Monetique-Eye Observability Platform**  
**Status**: Planning & Implementation  
**Goal**: Build and organize the `gitops/` directory to support one-click automated deployment from the backend.

## 1. Phase 4 Summary & Transition

Frontend and Backend are now complete and tested.  
Phase 5 focuses on preparing the **GitOps repository** so the `DeploymentService` can reliably execute SSH configuration, inventory generation, and Ansible playbooks.

This phase ensures:
- Clean separation between platform code and GitOps scripts
- Easy mounting inside Docker containers
- Full compatibility with your existing GitOps repo structure
- Secure and auditable deployment execution

## 2. Final GitOps Directory Structure

Create the following structure inside the root of `monetique-eye/`:
gitops/
├── vmpipe/                          # Central Node configuration (Docker Compose)
│   ├── docker-compose.yml
│   ├── prometheus/
│   ├── logstash/
│   └── grafana/
│
├── scripts/                         # Core deployment scripts (your existing ones)
│   ├── setup.sh
│   ├── deploy.sh
│   ├── update.sh
│   ├── ssh-configure.sh             # Critical for passwordless SSH
│   └── deploy-agent.sh
│
├── ansible/                         # Ansible playbooks and inventory
│   ├── inventory.ini                # Will be dynamically generated by backend
│   ├── deploy-tools.yml             # Deploys cAdvisor + Node Exporter + Filebeat
│   ├── deploy-backend.yml           # Optional: deploys Spring Boot backend
│   └── deploy-frontend.yml          # Optional: deploys React frontend
│
├── agents/                          # Agent node configurations
│   └── monitoring/                  # Templates for agent-side setup
│
└── README_GITOPS.md                 # Documentation for this directory
text## 3. Key Files to Prepare

### 3.1 scripts/ssh-configure.sh
Ensure this script exists and is executable. It should:
- Accept `AGENT_IP` and `AGENT_USER` as parameters
- Set up passwordless SSH from the central node to the agent

### 3.2 ansible/deploy-tools.yml
This is the most important playbook. It must:
- Install Node Exporter (port 9100)
- Install cAdvisor (port 8081)
- Install Filebeat (ships logs to Logstash on port 5045)
- Add environment label for Prometheus discovery

### 3.3 ansible/inventory.ini (Template)
The backend will dynamically generate this file before running Ansible.

4. Integration with Backend (DeploymentService)
In Phase 5 we ensure:

The gitops/ directory is mounted as a volume in the backend container
DeploymentService.java uses absolute paths like /app/gitops/scripts/ssh-configure.sh
Dynamic inventory is written to /app/gitops/ansible/inventory.ini before each run
All scripts are made executable (chmod +x)

5. Docker Volume Configuration (docker-compose.yml snippet)
Add this to your root docker/docker-compose.yml:
YAMLservices:
  backend:
    build: ../backend
    volumes:
      - ../gitops:/app/gitops:ro
    environment:
      - GITOPS_PATH=/app/gitops
