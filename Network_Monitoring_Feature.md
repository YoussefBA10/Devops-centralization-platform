    docker volume mount, and editing files in directories the user owns.

anything. All Ansible work targets the central VM only (writing config files,

             preferred_ip_protocol: "ip4"

           prober: tcp

       uri: http://localhost:9115/-/healthy
       expected status: 200

  Key metrics this produces (no new config on the VM — they already exist):
    node_netstat_Tcp_RetransSegs          — TCP retransmissions
    node_netstat_TcpExt_TCPFastRetrans    — fast retransmits (packet loss signal)
    node_netstat_TcpExt_TCPSynRetrans     — SYN retransmits (connection failure signal)
    node_netstat_Tcp_CurrEstab            — active TCP connections
    node_netstat_Tcp_ActiveOpens          — outgoing connection attempts/s
    node_network_receive_bytes_total      — inbound bandwidth per interface
    node_network_transmit_bytes_total     — outbound bandwidth per interface

    node_network_transmit_drop_total      — transmit packet drops
    node_network_receive_errs_total       — receive errors
    node_sockstat_TCP_tw                  — TIME_WAIT sockets (connection churn)

  IMPORTANT: Verify node_exporter was started with --collector.netstat and
  --collector.tcpstat flags. If not, these metrics will be absent. The backend

  node_exporter with those flags. Do NOT attempt to restart it automatically
  (no sudo access).

  CREATE TABLE vm_registry (
    id                  VARCHAR(36)  PRIMARY KEY,
    name                VARCHAR(100) NOT NULL,
    ip_address          VARCHAR(45)  NOT NULL,
    role                VARCHAR(50)  NOT NULL,    -- backend|frontend|db|gateway|proxy
    cluster_id          VARCHAR(36)  NOT NULL,
    env                 VARCHAR(20)  NOT NULL,
    node_exporter_port  INT          NOT NULL DEFAULT 9100,

    app_name            VARCHAR(100),
    netstat_collector_confirmed  BOOLEAN DEFAULT FALSE,  -- whether netstat metrics exist
    created_at          TIMESTAMP             DEFAULT CURRENT_TIMESTAMP,
    updated_at          TIMESTAMP             DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (cluster_id) REFERENCES cluster(id)
  );

  CREATE TABLE service_link (
    id              VARCHAR(36)  PRIMARY KEY,
    name            VARCHAR(200),                 -- e.g. "api-gateway → user-service"
    source_vm_id    VARCHAR(36)  NOT NULL,
    target_vm_id    VARCHAR(36)  NOT NULL,
    target_port     INT          NOT NULL,
    target_path     VARCHAR(200)          DEFAULT '/health',
    protocol        VARCHAR(10)           DEFAULT 'http',   -- http | tcp
    probe_module    VARCHAR(50)           DEFAULT 'http_2xx', -- http_2xx | tcp_connect
    enabled         BOOLEAN               DEFAULT TRUE,
    created_at      TIMESTAMP             DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_vm_id) REFERENCES vm_registry(id),
    FOREIGN KEY (target_vm_id) REFERENCES vm_registry(id)
  );

  CREATE TABLE network_alert_rule (
    id              VARCHAR(36)  PRIMARY KEY,
    name            VARCHAR(150) NOT NULL,
    rule_type       VARCHAR(50)  NOT NULL,
    -- LINK_DOWN | HIGH_LATENCY | CRITICAL_LATENCY | TCP_RETRANSMIT |
    -- PACKET_DROP | BANDWIDTH_SAT | HIGH_ERROR_RATE | HIGH_P99
    link_id         VARCHAR(36),    -- NULL = all links
    vm_id           VARCHAR(36),    -- NULL = all VMs
    threshold_value DOUBLE       NOT NULL,
    threshold_unit  VARCHAR(20),    -- ms | percent | per_second | Mbps
    severity        VARCHAR(20)           DEFAULT 'WARNING',  -- WARNING | CRITICAL
    enabled         BOOLEAN               DEFAULT TRUE,
    created_at      TIMESTAMP             DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (link_id) REFERENCES service_link(id),
    FOREIGN KEY (vm_id)   REFERENCES vm_registry(id)
  );

4.1 — ScrapeConfigGeneratorService

  Responsibilities:
    - Read all VmRegistry records from DB.
    - Read all ServiceLink records from DB.
    - Build a YAML string matching the scrape job templates in Part 2.
    - Validate the YAML is parseable before writing (use snakeyaml).
    - Write it to /opt/monitoring/prometheus/conf.d/network_monitor.yml.
      The path must be configurable via application.yml:
        monitoring.prometheus.conf-dir: /opt/monitoring/prometheus/conf.d
    - Call POST http://localhost:9090/-/reload with 5s connect timeout, 10s read timeout.
    - If reload returns non-2xx or times out, throw PrometheusReloadException with
      the response body included. The controller must return HTTP 500 with a clear
      message — never silently swallow this error.
    - This service is called automatically after any CRUD on vm_registry or service_link.
    - Writes are atomic: write to a .tmp file first, then rename to the final path.

4.2 — NetworkMetricsProxyService

  All queries target VictoriaMetrics (or Prometheus — configurable):
    monitoring.metrics.base-url: http://localhost:8428

  Time range to step mapping (use this exactly):
    15m  → step=3s,   start=now-15m
    1h   → step=12s,  start=now-1h
    6h   → step=72s,  start=now-6h
    24h  → step=288s, start=now-24h

  Methods to implement:

    getLinkMetrics(linkId, range):
      Queries:
        probe_success{link_id="<linkId>"}
        probe_duration_seconds{link_id="<linkId>"} * 1000          (convert to ms)
        probe_http_status_code{link_id="<linkId>"}
        probe_http_duration_seconds{phase="connect",link_id="<linkId>"} * 1000
        probe_http_duration_seconds{phase="processing",link_id="<linkId>"} * 1000
      Returns: { probeSuccess[], durationMs[], statusCode[], connectMs[], processingMs[] }

    getVmNetworkMetrics(vmId, range):
      Queries:
        rate(node_netstat_Tcp_RetransSegs{vm_id="<vmId>"}[5m])
        rate(node_network_receive_drop_total{vm_id="<vmId>"}[5m])
        rate(node_network_receive_bytes_total{vm_id="<vmId>"}[5m]) * 8 / 1e6    (Mbps)
        rate(node_network_transmit_bytes_total{vm_id="<vmId>"}[5m]) * 8 / 1e6   (Mbps)
        node_netstat_Tcp_CurrEstab{vm_id="<vmId>"}
        node_sockstat_TCP_tw{vm_id="<vmId>"}
        rate(node_network_receive_errs_total{vm_id="<vmId>"}[5m])
      Returns: { retransmitRate[], dropRate[], rxMbps[], txMbps[], tcpEstab[], timeWait[], errors[] }

    getVmContainerNetworkMetrics(vmId, range):
      Queries (from cAdvisor):
        rate(container_network_receive_bytes_total{vm_id="<vmId>"}[5m]) * 8 / 1e6
        rate(container_network_transmit_bytes_total{vm_id="<vmId>"}[5m]) * 8 / 1e6
        rate(container_network_receive_dropped_total{vm_id="<vmId>"}[5m])
      Group by container_label_com_docker_compose_service to identify per-container data.
      Returns: map of containerName → { rxMbps[], txMbps[], drops[] }

    getHealthSummary():
      For each enabled service_link, execute:
        avg_over_time(probe_success{link_id="<id>"}[1h])           → uptime %
        probe_duration_seconds{link_id="<id>"}                     → current latency
        histogram_quantile(0.50, rate(probe_duration_seconds_bucket{link_id="<id>"}[1h]))
        histogram_quantile(0.95, rate(probe_duration_seconds_bucket{link_id="<id>"}[1h]))
        histogram_quantile(0.99, rate(probe_duration_seconds_bucket{link_id="<id>"}[1h]))
      Note: If blackbox does not expose duration as a histogram (it exposes a gauge),
      use probe_duration_seconds directly for current value and avg/max over window
      instead of histogram_quantile. Adjust the query to:
        avg_over_time(probe_duration_seconds{link_id="<id>"}[1h])  → p50 approximation
        max_over_time(probe_duration_seconds{link_id="<id>"}[1h])  → p99 approximation
      Compute status:
        DOWN     = last probe_success == 0
        DEGRADED = probe_success == 1 AND (currentLatencyMs > 500 OR errorRate > 1%)
        UP       = otherwise
      Return: List<LinkHealthSummary> with { linkId, linkName, sourceVm, targetVm,
              status, currentLatencyMs, avgLatencyMs, maxLatencyMs, uptimePercent,
              errorRate, lastChecked }

    checkExporterCollectors(vmId):
      Query: node_netstat_Tcp_RetransSegs{vm_id="<vmId>"}
      If result is empty → return { netstatAvailable: false,
        warning: "node_exporter on this VM was not started with --collector.netstat.
                  TCP retransmission metrics are unavailable. To enable them, restart
                  node_exporter with the flags: --collector.netstat --collector.tcpstat.
                  Since you only have Docker group access, if node_exporter runs as a
                  systemd service, ask a user with sudo access to restart it with these
                  flags, or re-deploy it as a Docker container with the flags added." }
      If result has data → return { netstatAvailable: true }

4.3 — NetworkTopologyService

  buildTopologyGraph(clusterId, env):
    - Fetch all VmRegistry for cluster+env.
    - Fetch all ServiceLink where both VMs are in that cluster+env.
    - For each link, call getHealthSummary() to get current status + latency.
    - Return:
        {
          nodes: [{ id, label, role, ip, status, alertCount }],
          edges: [{ id, source, target, status, latencyMs, protocol, probeModule }]
        }
    - node.status = HEALTHY if no alerts firing, WARNING or CRITICAL otherwise.
    - node.alertCount = count of active Alertmanager alerts for that vm_id label.

4.5 — Prometheus alert rules file generator

  After any change to network_alert_rule table, regenerate:
    /opt/monitoring/prometheus/rules/network_alerts.yml
  Then reload Prometheus (same reload logic as scrape config).

  Default rules to seed in the DB migration (INSERT statements in V3__):

    LINK_DOWN:        probe_success == 0, for 1m,  CRITICAL
    HIGH_LATENCY:     probe_duration > 500ms, for 2m, WARNING
    CRITICAL_LATENCY: probe_duration > 2000ms, for 1m, CRITICAL
    TCP_RETRANSMIT:   rate(RetransSegs[5m]) > 10/s, for 3m, WARNING
    PACKET_DROP:      rate(receive_drop[5m]) > 5/s, for 2m, WARNING
    BANDWIDTH_SAT:    rx or tx > 800 Mbps, for 5m, WARNING
    HIGH_ERROR_RATE:  http 5xx rate > 1%, for 2m, WARNING
    HIGH_P99:         p99 > 500ms, for 3m, WARNING

  Generated rules file format:

    groups:
      - name: network_monitor
        interval: 30s
        rules:

          - alert: ServiceLinkDown
            expr: probe_success{job="blackbox_probe"} == 0
            for: 1m
            labels: { severity: critical }
            annotations:
              summary: "Link DOWN: {{ $labels.link_name }}"
              description: >
                Blackbox probe failed. Source: {{ $labels.source_vm }},
                Target: {{ $labels.target_vm }}, Link: {{ $labels.link_id }}.
                Last status code: check probe_http_status_code metric.

          - alert: HighServiceLatency
            expr: probe_duration_seconds{job="blackbox_probe"} > 0.5
            for: 2m
            labels: { severity: warning }
            annotations:
              summary: "High latency on {{ $labels.link_name }}"
              description: "{{ $labels.link_name }}: {{ $value | humanizeDuration }} (threshold 500ms)"

          - alert: CriticalServiceLatency
            expr: probe_duration_seconds{job="blackbox_probe"} > 2
            for: 1m
            labels: { severity: critical }
            annotations:
              summary: "Critical latency on {{ $labels.link_name }}"
              description: "{{ $labels.link_name }}: {{ $value | humanizeDuration }} (threshold 2s)"

          - alert: HighTCPRetransmissions
            expr: rate(node_netstat_Tcp_RetransSegs{job="node_exporter"}[5m]) > 10
            for: 3m
            labels: { severity: warning }
            annotations:
              summary: "TCP retransmissions elevated on {{ $labels.vm_name }}"
              description: "{{ $value | humanize }}/s retransmits on {{ $labels.vm_name }} ({{ $labels.vm_id }}). Possible packet loss or congestion."

          - alert: NetworkPacketDrops
            expr: rate(node_network_receive_drop_total{job="node_exporter"}[5m]) > 5
            for: 2m
            labels: { severity: warning }
            annotations:
              summary: "Packet drops on {{ $labels.vm_name }}"
              description: "{{ $value | humanize }}/s drops on {{ $labels.vm_name }}, interface {{ $labels.device }}"

          - alert: NetworkBandwidthSaturation
            expr: rate(node_network_receive_bytes_total{job="node_exporter"}[5m]) * 8 > 8e8
            for: 5m
            labels: { severity: warning }
            annotations:
              summary: "Bandwidth saturation on {{ $labels.vm_name }}"
              description: "Inbound: {{ $value | humanize }}bps on {{ $labels.vm_name }} (>800Mbps threshold)"

          - alert: HighHTTPErrorRate
            expr: >
              (
                rate(http_server_requests_seconds_count{status=~"5.."}[5m])
                /
                rate(http_server_requests_seconds_count[5m])
              ) > 0.01
            for: 2m
            labels: { severity: warning }
            annotations:
              summary: "HTTP 5xx rate >1% on {{ $labels.app_name }}"
              description: "Error rate: {{ $value | humanizePercentage }} on {{ $labels.app_name }} ({{ $labels.vm_id }})"

          - alert: AppP99LatencyHigh
            expr: >
              histogram_quantile(0.99,
                rate(http_server_requests_seconds_bucket[5m])
              ) > 0.5
            for: 3m
            labels: { severity: warning }
            annotations:
              summary: "p99 >500ms on {{ $labels.app_name }}"
              description: "p99={{ $value | humanizeDuration }} on {{ $labels.app_name }} ({{ $labels.vm_id }})"
